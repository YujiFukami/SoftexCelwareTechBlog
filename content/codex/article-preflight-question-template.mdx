## はじめに

AIエージェントに記事化を依頼すると、素材の技術的な説明は作れても、公開してよい範囲や読者に伝えるべき主眼までは判断できないことがあります。

特に実案件由来のコード、Excelブック、スクリーンショット、開発事例は、記事化の前に確認しておくべき項目があります。

この記事では、記事化前にユーザーへ確認する質問テンプレートを整理します。

## 使う場面

- 素材の業務目的がわからない
- 技術記事と開発事例のどちらにするか迷う
- 顧客名や案件名が含まれている可能性がある
- 画像をそのまま使ってよいかわからない
- 公式サイトのサービス導線とつなげたい

## 基本質問

まずは、素材全般に使える質問です。

```text
この素材について、記事化前に確認したいです。

1. このツール、ブック、コードは何の業務や目的で使うものですか？
2. 技術記事として公開したいですか、それとも開発事例として公開したいですか？
3. 顧客名、会社名、案件名、画面名など、公開してはいけない情報は含まれていますか？
4. 画像や動画をそのまま使ってよいですか？それとも図解を作り直した方がよいですか？
5. 読者に一番伝えたいポイントは、技術の仕組み、実装例、業務改善効果のどれですか？
```

## Excel/VBAブック向け質問

<Term>Excel VBA</Term>のブックでは、シート構成、マクロの役割、コード公開範囲を確認します。

```text
Excel/VBAブックについて確認したいです。

1. このブックの利用者は誰ですか？
2. 入力するシート、出力するシート、マスタシートはどれですか？
3. 一番重要なマクロはどれですか？
4. 汎用化できるプロシージャはありますか？
5. 記事にする場合、シート構成の図解を作った方がよいですか？
6. VBAコード全文を公開してよいですか？一部だけにするべきですか？
7. 顧客名や実データをサンプルデータに置き換えてよいですか？
```

## 開発事例向け質問

開発事例では、技術だけでなく、課題、作ったもの、改善効果、公開粒度を確認します。

```text
開発事例として整理するために確認したいです。

1. 依頼前の課題は何でしたか？
2. 作成したものは何ですか？
3. 使った主な技術は何ですか？
4. 導入後に何が改善されましたか？
5. 顧客名や業種は公開できますか？公開できない場合、どの粒度まで一般化しますか？
6. 公式ホームページから問い合わせにつなげる場合、どのサービス説明と関連付けたいですか？
```

## 図解・画像向け質問

画像は公開リスクが高いため、実画面を使うか抽象図にするかを先に確認します。

```text
図解を作るか確認したいです。

1. 図にしたい流れは、入力、処理、出力ですか？それともシステム構成ですか？
2. 実画面スクリーンショットを使いますか？抽象化した図にしますか？
3. 顧客名や実データを完全に除外してよいですか？
4. 画像生成用のプロンプトをこちらで作成してよいですか？
5. OGP画像、記事内説明図、開発事例図のどれに使いますか？
```

## 注意点

- 質問は必要な分だけ使い、毎回すべてを投げない
- ユーザーが明確に指示している範囲では、確認で止まりすぎない
- 公開リスクがある場合は、本文作成より先に確認する
- 判断できない固有名詞を見つけたら、サンプル名へ置換してよいか確認する

## まとめ

記事化前の質問は、作業を止めるためではなく、公開範囲と読者向けの主眼をそろえるために使います。

素材だけで判断できない場合は、業務目的、公開可否、画像利用、伝えたいポイントを先に確認すると、後から大きく書き直すリスクを減らせます。
