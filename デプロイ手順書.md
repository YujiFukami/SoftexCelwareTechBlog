# デプロイ手順書 - Softex Celware Tech Blog

サイト構築完了後、以下の手順で公開します。

---

## Step 1: GitHubリポジトリ作成

### 1-1. .gitignore の確認
`website/` フォルダ内に `.gitignore` が既にあります（create-next-appで自動生成済み）。
`node_modules/` 等は除外されています。

### 1-2. GitHubでリポジトリ作成
1. https://github.com/new を開く
2. 以下を入力：
   - **Repository name**: `softex-celware-tech-blog`
   - **Description**: Softex Celware Tech Blog - 技術ノウハウ公開サイト
   - **Public** を選択（Vercel無料枠はPublicリポジトリが必要）
3. 「Create repository」をクリック

### 1-3. ローカルからプッシュ
ターミナルで `website/` フォルダに移動して実行：

```bash
cd "c:/Users/fukam/OneDrive/d開発/20260415 Claude開発テクニック引継ぎ/website"
git init
git add .
git commit -m "Initial commit: Softex Celware Tech Blog"
git branch -M main
git remote add origin https://github.com/YujiFukami/softex-celware-tech-blog.git
git push -u origin main
```

> **注意**: GitHubのユーザー名は実際のものに置き換えてください

## Step 2: Vercelでプロジェクト作成

1. https://vercel.com/yujifukamis-projects を開く（既存アカウント）
2. 右上の **「Add New...」→「Project」** をクリック
3. **「Import Git Repository」** で先ほど作成したリポジトリ `softex-celware-tech-blog` を選択
4. 設定はデフォルトのままでOK（Next.jsが自動検出される）
5. **「Deploy」** をクリック
6. 数分でデプロイ完了 → `softex-celware-tech-blog.vercel.app` でアクセス可能に

---

## Step 3: カスタムドメイン設定（Vercel側）

1. Vercelでデプロイしたプロジェクトの画面を開く
2. 上部タブの **「Settings」** をクリック
3. 左メニューの **「Domains」** をクリック
4. 入力欄に `tech.softex-celware.com` と入力して **「Add」**
5. Vercelが以下のような案内を表示する：

```
Type: CNAME
Name: tech
Value: cname.vercel-dns.com
```

この値を控えておく（次のステップで使用）。

---

## Step 4: DNS設定（Wix側）

1. https://manage.wix.com/ にログイン
2. 左メニュー **「設定」** → **「ドメイン」** をクリック
3. `softex-celware.com` をクリック
4. **「DNSレコード」** タブ（または「DNSレコードを管理」）を開く
5. **「レコードを追加」** をクリック
6. 以下を入力：

| 項目 | 値 |
|------|-----|
| タイプ | **CNAME** |
| ホスト名 | `tech` |
| 値（ポイント先） | `cname.vercel-dns.com` |
| TTL | 3600（またはデフォルト） |

7. **「保存」** をクリック

### 反映確認
- 通常は数分〜1時間程度で反映
- 最大48時間かかる場合もある
- 反映後、`https://tech.softex-celware.com` にアクセスして確認
- SSL証明書はVercelが自動発行（HTTPS対応）

## トラブルシューティング

### ドメインが反映されない
- WixのDNS設定を再確認（ホスト名が `tech` になっているか）
- Vercelの Domains 画面で状態を確認（「Valid Configuration」と表示されるまで待つ）
- 48時間経っても反映されない場合は、WixとVercelの両方のサポートに問い合わせ

### ビルドエラー
- Vercelのデプロイログを確認（プロジェクト → Deployments → 失敗したデプロイをクリック）
- ローカルで `npm run build` が通るか確認してからpush

### 画像が表示されない
- `public/` フォルダに画像が含まれているか確認
- ファイル名の大文字小文字に注意（Vercelは大文字小文字を区別する）
