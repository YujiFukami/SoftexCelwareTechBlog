## 1. プロジェクト概要

### 1.1 目的
開発業務で蓄積した技術ノウハウを一般公開し、以下を達成する：

| 優先度 | 目的 | 詳細 |
|--------|------|------|
| **最優先** | 販路拡大・顧客誘導 | 技術記事を入口に、本体HP（softex-celware.com）への流入を増やし、開発依頼・相談に繋げる |
| **重要** | 技術的権威性の確立 | 「この人に頼めば解決してくれる」という信頼感を記事で醸成 |
| **副次的** | 開発者コミュニティへの貢献 | 他の技術者が参考にできるナレッジベースとして機能 |

### 1.2 ターゲットユーザー

| # | ターゲット | ペルソナ | 期待する行動 |
|---|-----------|---------|-------------|
| 1 | **中小企業の担当者** | DX・業務改善を進めたいが、ITに詳しくない。「Excel 自動化」「GAS 業務効率化」等で検索 | 記事で信頼 → 本体HPで相談・依頼 |
| 2 | **他の開発者・技術者** | 実装方法を探してググっている。Qiita/Zenn等も見る層 | 記事を参考にする → ブックマーク → サイトの認知拡大（被リンク・SNS拡散） |

### 1.3 本体サイトとの関係

```
[技術ブログ: tech.softex-celware.com（仮）]
  │
  │  記事を読む → 信頼感を得る
  │  CTAクリック → 本体HPへ誘導
  ↓
[本体HP: www.softex-celware.com]
  │
  │  サービス内容・実績・料金を確認
  ↓
[無料相談/打合せ予約]
```

### 2.3 各ページの要件

- ヒーローセクション: サイト名 + キャッチコピー + 本体HPへの導線

- 検索機能（記事タイトル・タグで検索）

#### 記事ページ（/articles/[category]/[slug]）
- **人が読みやすい記事形式**（現在のAI向けMDとは異なる構成）
  - 導入文（この記事で何が解決できるか）
  - 課題の説明（なぜこの技術が必要か）
  - 解決策の解説（コード付き、コピーボタン）
  - 実装のポイント・注意点
  - まとめ
- コードブロックにシンタックスハイライト
- コードのコピーボタン
- 目次（見出しから自動生成）
- カテゴリ・タグ表示
- 関連記事リンク
- **CTA: 本体HPへの誘導**（記事末尾）

#### Aboutページ（/about）
- 簡潔な自己紹介・事業紹介
- 対応可能な技術スタック
- 本体HPへのリンク（メインCTA）

## 3. デザイン要件

### 3.1 デザインの方向性
- 本体HP（softex-celware.com）と統一感のあるデザイン
- **青系を基調**とした配色（本体HPに合わせる）
- プロフェッショナルかつ親しみやすい雰囲気
- モバイルファースト（レスポンシブ対応必須）

### 3.2 本体HPへの誘導UI

| 配置場所 | UI要素 | 内容 |
|---------|--------|------|
| **ヘッダー（常時表示）** | ロゴ + 小さなリンク | 「Softex Celware」ロゴ → 本体HPリンク。控えめだが常に見える |
| **記事末尾（CTA）** | バナー風セクション | 「この技術で業務改善しませんか？ → 無料相談はこちら」本体HPへリンク |
| **サイドバー or フッター** | ミニプロフィール | 事業紹介（1-2行）+ 本体HPリンク。くどくならない程度 |

**設計方針**: 誘導は「押し付けない」が「見つけやすい」バランス。
記事の価値を損なわず、自然な流れで「この人に依頼できるんだ」と気づかせる。

## 4. 技術要件

### 4.1 技術スタック

| 項目 | 技術 | 理由 |
|------|------|------|
| フレームワーク | **Next.js（App Router / SSG）** | 既存スキル活用。SSGでSEO最適 |
| スタイリング | **Tailwind CSS** | 高速開発。レスポンシブ対応が容易 |
| コンテンツ管理 | **MDXファイル（ローカル）** | 既存MDファイルとの親和性。CMS不要 |
| MD処理 | **next-mdx-remote** or **contentlayer** | Markdownを記事ページに変換 |
| デプロイ | **Vercel** | Next.jsとの相性最良。無料枠で十分 |
| ドメイン | **tech.softex-celware.com**（サブドメイン） | 本体ドメインのSEO資産を活用 |
| バージョン管理 | **GitHub** | Vercelとの自動連携デプロイ |

### 4.2 SSG（静的サイト生成）の採用理由
- 各記事が静的HTMLとして生成 → 表示速度が極めて速い
- Google検索のクロールに最適（SEO有利）
- サーバーコスト不要（Vercel無料枠で運用可能）
- セキュリティリスクが低い（データベース不要）

## 5. SEO要件（内部SEO）

### 5.1 技術的SEO

| 項目 | 対応内容 |
|------|---------|
| メタタグ | 各記事ごとにtitle / description を自動生成 |
| OGP画像 | 記事タイトルから動的にOGP画像を自動生成（SNS共有時に表示） |
| 構造化データ | JSON-LD で `TechArticle` / `HowTo` スキーマを埋め込む |
| sitemap.xml | 記事追加時に自動生成 |
| robots.txt | 適切なクロール許可設定 |
| canonical URL | 重複コンテンツ防止 |
| パンくずリスト | カテゴリ階層を明示（構造化データ付き） |

### 5.2 コンテンツSEO

| 戦略 | 詳細 |
|------|------|
| **キーワード設計** | 開発者が実際に検索するフレーズをタイトル・見出しに含める（例:「GAS ローディング CSS」「Supabase 1000行 制限 対処」） |
| **記事構成** | 「課題 → 解決策 → コード → ポイント」の一貫した構成で滞在時間を伸ばす |
| **内部リンク** | 関連記事同士をリンクし、サイト内回遊を促進 |
| **更新頻度** | 高頻度（開発のたびに蓄積 → 記事化）でGoogleの評価を維持 |
| **URLスラッグ** | 英語で簡潔に（例: `/articles/gas/css-spinner`） |

## 6. 記事コンテンツの変換ルール

### 6.1 現在のMDファイル（AI向け）→ 公開記事 への変換

**変換前（AI向け・現在の形式）:**
```markdown
# CSSスピナー（ローディングアニメーション）
## 概要
GASでは外部画像やCDNの読み込みが面倒なため...
## コード
（コードブロック）
## カスタマイズ
```

**変換後（人間向け・公開記事）:**
```markdown
---
title: "GASでCSSだけのローディングスピナーを実装する方法"
category: "gas"
tags: ["GAS", "CSS", "ローディング", "UI"]
description: "Google Apps Scriptで画像不要・CSSだけで回転スピナーを実装。コピペですぐ使えるコード付き。"
date: "2026-04-15"
---

## はじめに
GAS（Google Apps Script）でWebアプリを作っていると、
データ読み込み中にユーザーに「処理中」を伝えたい場面がありますよね。

外部CDNやライブラリを使う方法もありますが、GASの制約上...
（以下、読者に語りかける形で解説）

## こんな場面で使えます
- スプレッドシートから大量データを取得する間の待機表示
- フォーム送信後の処理待ち
- ...

## 実装コード
（コード + 解説）

## カスタマイズ例
（具体例 + ビフォーアフター的な説明）

## まとめ
（要点整理 + 関連記事リンク）
```

### 6.2 記事の文体・トーン
- **ですます調**（丁寧だが堅すぎない）
- **読者に語りかける形**（「〜ありますよね」「〜してみましょう」）
- **具体的な利用シーンを提示**（「こんな場面で困っていませんか？」）
- **コードは動くものをそのまま載せる**（コピペで使える状態）
- **専門用語には簡単な補足を添える**（非エンジニアも一部読む想定）

## 12. デプロイ手順（サイト構築完了後に実施）

### Vercel側
1. Vercel ダッシュボード → 「Add New」→「Project」
2. GitHub リポジトリを連携
3. プロジェクト Settings → Domains → `tech.softex-celware.com` を追加
4. 表示されるCNAME値を控える

### Wix DNS側
1. Wix ダッシュボード → 設定 → ドメイン
2. `softex-celware.com` → DNS レコード管理
3. CNAMEレコード追加: ホスト名 `tech` → 値 `cname.vercel-dns.com`
4. 保存後、数分〜最大48時間で反映
